# 검색 증강 생성(RAG)

검색 증강 생성(Retrieval-Augmented Generation, RAG)은 2020년 Lewis 등이 제안했다.
외부 문서를 검색해 LLM의 생성에 근거를 붙이는 방식이다. 환각을 줄이고 최신 정보를 반영한다.

표준 RAG 파이프라인은 청킹, 임베딩, 벡터 검색, 생성 네 단계로 이뤄진다.
검색 단계가 약하면 생성 단계도 무너진다. 이 약점을 보완하려고 여러 후속 기법이 나왔다.

RAG의 한계는 여러 문서에 흩어진 사실을 엮는 멀티홉 질문에서 두드러진다.
이 문제의식이 GraphRAG 계열 연구로 이어졌다.
