# 집필 품질 게이트 (Authoring Checklist)

초안 작성 후, 그리고 윤문 후 반드시 통과해야 하는 검증 항목. kb-agent는 작성 직후 A·B를, 스킬은 윤문 후 C를 점검한다.

## A. 템플릿 6요소 충족 (lesson.md)

- [ ] **학습 목표 3개 이상** — 상단, 측정 가능한 행동 동사
- [ ] **완료 기준 1줄** — 명확하고 검증 가능
- [ ] **이론 + 코드 실습** — 개념 직후 실행 가능한 코드(이론만 나열 금지)
- [ ] **🚨 자주 하는 실수 1–3개** — 하단, 실제 함정 위주
- [ ] **출처** — 공식 docs/GitHub/논문 URL
- [ ] **다음 토픽 링크** — 마지막 줄, 상대 경로

## B. 구조·일관성·코드

- [ ] 디렉토리 = `course/phase-<NN>-<slug>/<NN>-<topic-slug>/` + `practice/` + `labs/`(+ 필요 시 `img/`)
- [ ] **슬러그·번호가 [study-roadmap.md]와 정확히 일치** (임의 변경·오타 없음)
- [ ] 내부 링크는 **상대 경로**, 이미지는 `./img/`
- [ ] 용어 표기가 [glossary.md]를 따름 (지식그래프·GraphRAG·멀티홉·LightRAG 5모드 등)
- [ ] 도구·버전·코퍼스가 [stack-conventions.md]를 따름. 임의 라이브러리 도입 없음
- [ ] 난이도가 [style-and-difficulty.md]의 페르소나에 맞음. 앞 Phase 산출물을 입력으로 가정
- [ ] **정적 검증 통과**: Python 문법 오류 없음, import 일관, 타입·변수 흐름 모순 없음
- [ ] 외부 의존(Neo4j·API 키·Docker) 코드에 전제 주석 + 비용 대안 분기 명시
- [ ] `labs/` 각 명령에 **예상 출력** 동반

## C. 윤문 후 보존 검증 (humanize-korean 실행 직후 — 최우선)

윤문은 **문체·리듬·표현만** 바꾼다. 아래가 한 글자라도 바뀌면 해당 구간을 롤백한다.

- [ ] **코드블록** — 윤문 전/후 완전 동일 (들여쓰기·변수명·주석 포함)
- [ ] **명령어·경로·파일명** — 동일
- [ ] **URL·출처** — 동일
- [ ] **수치·버전·날짜** — 동일 (`voyage-3.5`, Neo4j 5.26, arXiv 번호 등)
- [ ] **완료 기준 1줄** — 윤문 대상에서 제외, 원문 유지
- [ ] **영어 약어** — 원형 보존 (RAG·LLM·KG·GDS·API 등)
- [ ] **슬러그·토픽 번호** — 동일
- [ ] 변경률이 비정상적으로 높지 않음(산문 문체 수준의 변화여야 함)

### 보존 검증 방법

1. 윤문 전 lesson.md를 백업(예: `lesson.draft.md` 또는 메모리 보관).
2. 윤문 후 코드블록·URL·수치만 추출해 1:1 비교(diff).
3. 차이가 발견되면 해당 구간만 원문으로 되돌린다. 산문은 윤문본 유지.

> humanize-korean은 "수치·고유명사·직접 인용은 탐지/윤문 대상 아님"을 철칙으로 하지만, 강의자료는 코드·명령어·완료기준까지 보존 대상이 넓으므로 **스킬이 윤문 호출 시 보존 범위를 명시**하고, 호출 후 이 체크리스트로 재확인한다.
