# 문체·난이도 가이드 (Style & Difficulty)

모든 토픽이 **같은 독자에게, 같은 난이도 곡선으로, 같은 목소리로** 읽히게 만드는 기준이다.

## 대상 독자 (페르소나 고정)

> 개발·RAG·Agent **기본기는 있지만**, 최신 지식베이스/그래프 개념과 구축 방법을 제대로 배우고 싶은 개발자.

이 한 줄에서 난이도가 결정된다.

**전제로 깔아도 되는 것** (다시 설명하지 않는다)
- Python 3.11+ 문법, 가상환경, `pip`/`uv`, 비동기 기초
- Docker / docker compose 기본 사용
- LLM API 호출 경험, 프롬프트 기초, 임베딩·벡터 검색의 *개념*
- 기본 RAG 파이프라인(청킹 → 임베딩 → 벡터 검색 → 생성)을 만들어 본 경험
- Git, REST/JSON, SQL 기초

**반드시 처음부터 쌓아 주는 것** (이 과정의 핵심)
- 그래프 데이터 모델(LPG)·Cypher·Neo4j 운영
- 텍스트 → 엔티티·관계·클레임·이벤트 추출과 **정제**(Entity Resolution·품질 게이트)
- GraphRAG 검색 패턴(Local·Global·Path·Community)과 LightRAG 5모드
- 온톨로지·시맨틱 레이어, 평가·관측성, 에이전트 하니스

판단 기준: **"RAG는 아는데 그래프는 처음"** 인 사람이 막히지 않고, 동시에 지루하지 않게.

## 난이도 일관성 규칙

1. **앞 Phase 산출물을 입력으로 가정한다.** 누적 스토리라인(원문 → Wiki → KG → Neo4j → GraphRAG → Agent)을 깨지 않는다. 이전 토픽에서 만든 것을 다시 처음부터 만들지 않는다.
2. **개념 1개당 코드 1개.** 새 개념을 설명하면 곧바로 실행 가능한 최소 예제로 잇는다(이론만 나열 금지).
3. **새 도구·라이브러리는 첫 등장 시 1–2문장으로 "왜 이걸 쓰는지"** 를 밝힌다. 설치 명령은 practice에 둔다.
4. **점프 금지.** 한 토픽 안에서 다루지 않은 함수·개념을 코드에 슬쩍 쓰지 않는다. 필요하면 짧게 짚고 넘어간다.
5. **비용·환경 부담을 항상 의식한다.** 상용 API가 필요한 지점에는 로컬·무료 대안 분기를 [stack-conventions.md](stack-conventions.md)에 따라 명시한다.

## 설명 흐름 (모든 토픽 공통 골격)

```
왜 필요한가(동기·실패 사례)  →  핵심 개념(직관 우선, 정의는 짧게)
  →  최소 코드로 직접 해보기  →  결과 해석  →  흔한 실수  →  다음으로 잇기
```

- **동기 먼저.** "기존 방식이 여기서 무너진다 → 그래서 이걸 배운다" 구조를 유지한다(Phase 0의 RAG 실패 4종이 전체 동기).
- **직관 → 정의 → 코드** 순서. 정의를 먼저 던지고 시작하지 않는다.
- 다이어그램이 이해를 크게 돕는 경우 `./img/`에 둘 위치만 표시하고 본문은 텍스트로도 성립하게 쓴다.

## 문체 가이드 (humanize 친화 — 초안부터 AI 티를 줄인다)

윤문은 [humanize-korean] 스킬이 맡지만, **초안부터 아래를 지키면 윤문 부담이 준다.**

- **문장은 짧게, 길이를 의도적으로 들쭉날쭉하게.** 모든 문장이 비슷한 길이면 기계 티가 난다.
- **접속사("그리고·또한·하지만·따라서") 남발 금지.** 문장 관계로 흐름을 만든다.
- **기계적 병렬·3종 세트 나열 자제.** "A하고, B하고, C한다" 패턴 반복 금지.
- **이모지·불릿 과다 금지.** 불릿은 정말 목록일 때만. 설명은 문단으로.
- **번역투 회피**: "~에 대해", "~을 통해", "~라고 할 수 있다", "~하는 것이 중요하다" 남용 금지.
- **메타 문장 자제**: "이 절에서는 ~을 살펴본다" 같은 군더더기 도입부 최소화.
- **단정적으로.** "~일 수 있습니다"의 반복 대신 필요한 곳에서 분명히 말한다.
- **존댓말/평어 register는 한 토픽 안에서 통일**한다(기본: 해라체·평서형 설명 + 지시문은 자연스러운 종결).

## 절대 윤문하지 않는 것 (초안·윤문 공통 보존 대상)

코드블록, 명령어, 파일·경로명, URL, 수치·버전, 완료 기준 1줄, 출처 인용, 영어 약어. 자세한 보존 규칙은 [authoring-checklist.md](authoring-checklist.md).
